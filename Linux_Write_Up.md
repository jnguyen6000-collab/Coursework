
  The purpose of file permissions is to give or restrict access to the user’s ability to read, write, or execute the file. The importance of file permissions is to prevent the content within the file from being in the wrong hands by reducing attack vectors and making sure that only authorized users can access it. 

  Some command-line skills that help cybersecurity professionals detect and respond to OS attacks are commands such as find, grep, or piping which search for anything within the file that has the term they are looking for specifically. This search can help professionals narrow down potential vulnerabilities in the OS. With find or grep command, they can also save time looking for potential OS attacks. The ls \-l and chmod allow cybersecurity professionals to look at and change the permissions on a file or directory. If a file has too many permissions, they can take action to remove those permissions away from unauthorized users. There is also the cat command that reads the content in the directory. This enables cybersecurity professionals to check to make sure the content in the directory does not contain malware. 

  	An attacker might use command-line tools to compromise a system by using the ls \-1 command to see if they have unauthorized access. If they do, they can ruin the content of the file completely. 
    By understanding how these tools work, we take immediate action to defend by adding proper security measures to ensure it will not happen again. 
    An example would be to revoke access from unauthorized users by removing their permissions.
